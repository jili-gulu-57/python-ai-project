# 企业知识库多轮智能问答系统整合说明

## 项目定位

本项目将原来的「RAG 智能问答服务」和「AI 多轮聊天对话服务」整合为一个企业级知识库智能助理。

整合后的系统不再只是单次知识库问答，也不只是普通大模型聊天，而是支持：

- 多轮会话管理
- 基于历史上下文的问题改写
- 知识库向量检索
- RAG 检索增强回答
- 知识库片段与来源展示
- 普通聊天与知识库问答模式切换
- 本地会话保存与历史会话恢复
- 文本知识文件上传、切分、向量化入库

## 合并后的核心链路

```text
用户问题
  -> Streamlit 多轮聊天界面
  -> 读取当前 session 历史消息
  -> Query Rewrite：把省略、指代问题改写成完整检索问题
  -> Chroma 向量库召回知识片段
  -> Prompt 注入参考资料和历史对话
  -> 大模型生成有依据的回答
  -> 展示回答、保存会话、展示本轮检索片段
```

## 新增和修改的文件

- `app_integrated.py`
  - 新的整合版应用入口。
  - 包含知识库上传、会话管理、RAG 问答、普通聊天、检索详情展示。

- `rag.py`
  - 增加历史感知检索能力。
  - 增加 `rewrite_query()`，用于把多轮追问改写成完整检索问题。
  - RAG 回答 Prompt 增加来源引用和资料不足时拒答要求。

- `项目整合说明.md`
  - 当前说明文档，可直接作为项目 README 的基础版本。

## 推荐演示流程

1. 启动 `app_integrated.py`。
2. 在左侧上传一个 UTF-8 编码的 `.txt`、`.md` 或 `.csv` 文件。
3. 点击「写入知识库」。
4. 在「知识库问答」模式下提问，例如：“这个系统支持哪些功能？”
5. 继续追问：“那第二个功能怎么实现？”
6. 展开「本轮检索详情」，展示系统如何把多轮问题改写并检索知识片段。
7. 切换到「普通聊天」模式，演示普通多轮聊天能力。
8. 新建会话或加载历史会话，展示会话管理能力。

## 简历写法

项目名称：

> 基于 RAG 的企业知识库多轮智能问答系统

项目描述：

> 基于 Streamlit、LangChain、Chroma 和通义千问兼容 OpenAI API 构建企业知识库多轮智能问答系统，实现文档上传、文本切分、向量化存储、语义检索、多轮上下文管理、Query Rewrite、RAG 检索增强生成和答案来源展示。系统支持普通聊天与知识库问答双模式，能够解决多轮对话中指代、省略导致的检索不准确问题，并通过参考资料约束和低置信度拒答降低大模型幻觉风险。

技术亮点：

- 设计 Query Rewrite 链路，将历史对话和当前问题融合为完整检索问题，提升多轮追问场景下的召回准确率。
- 基于 Chroma 构建本地向量知识库，支持知识文件上传、切分、Embedding 和相似度检索。
- 使用 LangChain `RunnableWithMessageHistory` 管理不同用户会话，实现会话级短期记忆。
- 在回答 Prompt 中注入检索片段和来源信息，实现基于知识库的可溯源回答。
- 提供 Streamlit 可视化交互界面，支持历史会话管理、模式切换和检索过程展示。
